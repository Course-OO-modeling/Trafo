Detailed Description

Algorithmic realization of Translation and the Cardan's Rotation.

The method takes all needed parameters from the class ControlParms that extends the class EllipsoidParms. Needed parameter for the Transformation: 3 translation-, 1 scale- and 3 rotation -parameters. The start coordinates are taken from the class GeographicCoordinate and have to be converted into cartesian coordinates to realize the transformation. The Cardan's Rotation is done with trigonometric funtion. After the Transformation, the new coordinates are converted into ellipsoidal coordinates by an iterative process.

source of cardan's rotation: http://www.itm.uni-stuttgart.de/courses/madyn/Merkblaetter/M08.pdf (in Germany) Concept explanation : http://n.ethz.ch/~roclaudi/download/4.Semster/glossarium%20h%F7ge.pdf

void datumstrategy.CardansTransformation.transform ( GeographicCoordinateInterface  geo)

Use Cardan's Rotation to transform the geographic coordinate.

Translation : Cartesian coordinates are used. Rotation : rotate with principle of conventional Cardan's Rotation (Wx,Wy,Wz). Algorithm explanation : Rotate first about X-axis ,after first step rotate about current Y-axis ,at end rotate about current Z-axis.
